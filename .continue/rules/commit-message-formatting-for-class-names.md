---
description: This rule applies when writing commit messages for changes
  involving class names, file names, or module names. Formatting characters like
  backticks or quotes can cause rendering issues in commit logs, making
  filenames or class names disappear or appear incorrectly.
alwaysApply: false
---

Do NOT surround class names, file names, or module names with backticks, quotes, or any other formatting characters in commit messages. Use plain text to ensure clarity and avoid rendering issues.
